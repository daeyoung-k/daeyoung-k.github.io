## 주제

>예시로 member 디렉토리에 Member 클래스를 생성 후 테스트를 진행하고 해당 클래스를 삭제 합니다.  
>memebers 디렉토리에 member 클래스를 생성 하고 테스트를 진행하면 중복 bean 오류가 발생합니다.  
>이 오류를 해결하는 방법을 알아보겠습니다.

## 내용

인텔리제이 설정에 들어가 봅니다.  
`Build, Execution, Deployment / Build Tools / Gradle` 으로 접속해보면  
Build and Run 에 사용할 빌드 종류를 선택할수 있습니다.  

- Intellij IDEA 로 선택하고 프로그램을 실행하게 된다면 `out` 이라는 폴더가 생기면서 빌드된 파일들이 모이게 됩니다.
- Gradle 로 선택하고 프로그램을 실행하게 된다면 `build` 이라는 폴더가 생기면서 빌드된 파일이 모이게 됩니다. 

각 폴더에 생성된 파일을 기반으로 프로그램이 실행되는데 해당 폴더에 중복된 클래스가 생성되어 있어서 오류가 발생하게 됩니다.  

1. 파일 -> Invalidate Caches 로 캐쉬를 제거 진행.
2. 캐쉬를 제거해도 오류가 지속될때 
- Intellij 로 실행한다면 `out` 을 Gradle 로 실행한다면 `build` 폴더를 삭제하고 실행하면 오류가 없어집니다.
